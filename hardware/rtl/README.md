# Hand-written RTL

SystemVerilog and Verilog sources for the hand-written HDL accelerator and the
top-level wrappers that multiplex the HDL and HLS backends.

## Module index

### Top-level / integration

| File | Role |
|------|------|
| `tinyissimoyolo_accelerator.sv` | Packaged IP top (AXI-Lite + optional S_AXIS), wraps `inference_top` |
| `inference_top.sv`              | Dual-engine wrapper: instantiates `inference_hdl` + `inference_hls`, multiplexes URAM ports, exposes AXI-Lite register/result read-back |
| `axil_regs.sv`                  | AXI-Lite control/status register bank (CTRL, STATUS, MODE, INFO, result window, `engine_sel`) |

### HDL inference engine

| File | Role |
|------|------|
| `inference_hdl.sv`  | Multi-layer HDL inference FSM — per-layer loop over `LAYER_CFG`, dispatches `conv3d` / `conv1d`, feeds `max_pool` / `activation`, requantizes and writes back to URAM |
| `conv3d.v`          | 3D convolution engine (3×3 kernel cone) for feature-map layers |
| `conv1d.sv`         | 1×1 convolution engine for the detection head (`cv2` / `cv3`) |
| `convolver.v`       | Generic MAC cone used by `conv3d` |
| `mac_manual.v`      | Hand-instantiated DSP48 MAC unit |
| `max_pool.sv`       | 2×2 max pooling stage (layer-by-layer gated by `LAYER_CFG.has_pool`) |
| `activation.sv`     | SiLU LUT-based activation, packed 128-entry int8 table |

### Memory primitives

| File | Role |
|------|------|
| `circular_buffer.v` | Sliding-window line buffer used by `conv3d` |
| `sdp_ram.sv`        | Simple dual-port RAM wrapper — behavioural model for sim, maps to URAM288 in synth |

### HLS engine wrapper

| File | Role |
|------|------|
| `inference_hls.sv`  | Thin SystemVerilog wrapper around the Vitis-HLS `tinyissimo_layer_top` IP. Handles layer-0 RGB unpack, drives `ap_start` / `ap_done`, exposes the same URAM-port interface as `inference_hdl` so `inference_top` can mux between engines |

### HLS-exported sources (auto-generated)

`tinyissimo_layer_top*.v` and `tinyissimo_layer_top_*_ROM_*.dat` files are
copies of the Vitis HLS synthesis output mirrored from `hardware/hls/` via
`hardware/scripts/sync_hls_to_rtl.sh`. They are not hand-edited — regenerate
from the HLS source instead.

## Regenerating weight-dependent code

`hardware/weights/hdl/layer_config.svh` is generated by
`hardware/scripts/generate_hdl_weights.py` from the canonical TFLite model. Do
not edit it by hand — see
[`hardware/scripts/README.md`](../scripts/README.md) for the full pipeline.

Inference Time

  • TFLite: wall-clock of interpreter.invoke()
  • HDL / HLS: cycle counter (register 0x00C) converted to ms at the 100 MHz PL clock — pure on-FPGA compute time, bus transfer overhead excluded
  • Thread dropdown on the TFLite row lets you compare 1 / 2 / 4 CPU threads

Inference Cycles

  • HDL / HLS only — the exact number of 10 ns PL cycles spent computing this image
  • Read from the hardware cycle counter after STATUS.done asserts

End-to-End Latency

  • Pre-Process + Inference Time + Post-Process for a single image
  • HDL host wall-clock is ~150 ms higher than this number — the gap is the AXI-Lite pixel preload (65 K writes over the slow control bus)
  • That overhead is bus protocol cost, not inference cost, so the table excludes it to keep the comparison fair

FPS (Inference) vs FPS (End-to-End)

  • FPS (Inference): apples-to-apples engine throughput — 1000 / Inference Time
  • FPS (End-to-End): what a single-image loop observes in practice — 1000 / E2E Latency

Live Slider Updates

  • Confidence and NMS IoU are post-processing filters — they don't change the model's raw output tensor
  • After Run Inference, the raw tensor is cached; slider drags re-run only post-processing
  • Pre-Process, Inference Time, and Cycles stay frozen; Post-Process / E2E / FPS update live

NMS IoU is Counterintuitive

  • Controls how much overlap is tolerated before two boxes are called duplicates
  • Higher values keep more boxes (fewer pairs exceed the threshold)
  • 1.0 = no suppression; 0.0 = merge everything that overlaps
  • Default 0.45 follows the YOLO / OpenCV convention

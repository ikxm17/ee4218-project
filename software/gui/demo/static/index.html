Inference Time
For TFLite, the wall-clock of interpreter.invoke(). For the HDL accelerator, the cycle counter (register 0x00C) converted to milliseconds at the 100 MHz PL clock — i.e. the pure on-FPGA compute time, with bus transfer overhead excluded.
Inference Cycles
HDL only. The exact number of 10 ns cycles the accelerator spent computing this image, read from the cycle counter after STATUS.done asserts.
End-to-End Latency
Pre-Process + Inference Time + Post-Process for a single image. For HDL the host driver wall-clock is ≈ 150 ms higher than this latency number — the difference is the AXI-Lite pixel preload (65 K writes over the slow control bus). That overhead is bus protocol cost, not inference cost, so the table excludes it from the End-to-End column to keep the comparison fair. The actual measured host wall-clock is reported in the model-info panel under Hardware.
FPS (Inference) vs FPS (End-to-End)
FPS (Inference) is the apples-to-apples engine throughput (1000 / Inference Time). FPS (End-to-End) is what a naive single-image loop would see in practice (1000 / End-to-End Latency).
Live Slider Updates
Confidence and NMS IoU are pure post-processing filters — they don't change the model's raw output tensor. After you click Run Inference, the raw tensor is cached and slider drags re-run only the post-process stage on the cached tensor. That's why Pre-Process, Inference Time, and Inference Cycles cells stay frozen while Post-Process / End-to-End / FPS update live.
NMS IoU is Counterintuitive
The NMS IoU slider controls how much overlap is tolerated before two boxes are called duplicates. Higher values therefore keep more boxes, not fewer — because fewer pairs exceed the threshold. Dragging to 1.0 disables suppression entirely (no two boxes have IoU > 1). Dragging toward 0.0 is the aggressive "merge everything that overlaps at all" end. Default 0.45 is the standard YOLO/OpenCV convention. The endpoint labels merge all / keep all next to the slider indicate this direction.
